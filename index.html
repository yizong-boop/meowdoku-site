Product requirement · 研发蓝图

把每一步推理，
放进猫咪的棋盘里。

Meowdoku 是一款猫咪主题的轻量逻辑拼图。玩家在不同尺寸的棋盘上放置猫咪，逐步满足行、列、相邻和颜色区域规则；主线负责成长，Daily 负责每天回来。

02 · Core loop

主玩法：每一步都可解释

核心目标不是让玩家“试出来”，而是让每次正确放置都能被规则支持。错误应当有反馈，但不能让玩家迷失。
P0 · 必须稳定
⌁
本版实现基线动态难度和连击已按 APK 静态代码重新核对。页面中的“已确认”只代表代码/资源可证明；远程 AB 实际命中组、音效时序和设备表现仍需真机或线上配置验证。

2.1 一局游戏怎么发生 #

Home→Play Level→棋盘加载→放置 / 标记 / 道具→Win / Fail→下一关或恢复

玩家要做什么

  • 在每一行放置一只猫，最终总数等于棋盘尺寸。
  • 用行、列、相邻和区域关系排除不可能位置。
  • 生命耗尽进入失败页；完成所有行进入完成页。
  • 退出后再次进入，继续当前残局而不是从零开始。

棋盘规则

点击格子：放置猫咪；再次点击：移除。颜色代表不同区域。
✦
研发交互原则错误放置要明确告诉玩家“为什么不行”，但不要把错误变成复杂惩罚。主玩法的节奏应该来自推理，而不是反复试错。

2.3 动态难度与题库 #

策略

不是固定关卡映射

代码持久化 current_strategy，再按进度上限、AB 随机和 Hard/Special 分支选择 rank/tier。

升档

看上一关完成状态

6—50 关需连续 2 次 clean win；51 关后每次 clean win 可升档，均不超过当前阶段上限。

降档

失败阈值可确认

6—20 关累计 1 次失败，21 关以后累计 2 次失败；DDA 额外模式再处理重试/道具/复活。

⌁
证据边界当前 APK 能确认算法和代码默认值，但不能确认远程 AB 最终命中的分组。Hard 默认从 21 关开始每 10 关一次；GROUP_J 实验组使用另一套 Hard 节点。题目数据包含区域地图、解答、难度和提示参考值，进入游戏前应完成合法性检查。

2.4 道具、草稿和自动标记 #

Hint

给出一步推理

高亮当前可理解的下一步，不直接展示整局答案。

Locate

定位正确位置

揭示一个可放置的正确位置，帮助玩家继续游戏。

Undo

撤销最近操作

普通关和 Daily 共享库存，AB 可配置为免费。

草稿模式

中后段开放。玩家记录候选位置，支持冲突检测、提交、清除和按配置恢复。

自动标记

根据已放置猫咪自动标记不可用位置，首次使用展示引导；Daily 可配置为激励视频解锁。

!
与动态难度的关系静态代码明确将 Hint、Locate、返回 Home / Restart 标记为 dirty；因此不能把“使用道具是否影响 clean win”写成完全由 AB 决定。Undo 的 dirty 影响在当前已导出代码中未发现明确标记，需真机或进一步运行时验证。

2.5 生命、连击和结算 #

生命系统

普通关和 Daily 默认进入 3 条生命；错误放置扣 1 条，耗尽进入 Fail。revive_life 控制组恢复 1 条，其他已定义组恢复 3 条，均封顶 3 条。

连击反馈

3 Nice4 Great5 Perfect6 Excellent7 Amazing8+ Unbelievable

正确猫咪每次 +1；错误立即清零。代码从 3 次开始播放文字动画，视觉素材与音频素材均已在资源站对应。

Score

局内累计分数

每次正确放置都增加分数并显示气泡。普通 Score 模式：min(600 + max(combo−1,0)×80, 1320)。

IQ

另一种数值表现

IQ 模式每次增加 min(6 + max(combo−1,0)×2, 24)；顶部从展示基线 40 开始。

Follow Cat

跟随猫咪展示

仍使用普通 Score 增量，但 Nice～Unbelievable 文案跟随最近放置的猫显示。错误只清连击，不扣已累计分数。

Σ
分数与完成百分比分离当前代码只证明分数用于局内反馈、连击气泡和残局恢复；没有证据证明它参与普通关“打败 xx%”、Daily 成绩或在线排行榜。
1

Win 完成页

展示完成文案、庆祝动效、用时、步数、最高连击和表现等级；普通关进入下一关，Daily 返回 Daily。

2

Fail 失败页

提供复活、重新开始和返回。复活成功要恢复当前棋盘状态，广告失败不能阻断返回和重玩。

3

残局恢复

保存棋盘、猫咪位置、标记、生命、步数、连击和复活状态；异常退出后不丢奖励、不重复发奖。

结算百分比不是排行榜

普通关的“打败 xx%”由本地用时模型生成，不查询真实玩家。基础输入是 elapsed_sec 和棋盘尺寸；尺寸 P90 秒数为 4×4=44、5×5=37、6×6=107、7×7=187、8×8=265、9×9=360、10×10=431。

beat = 51 + 48 × sqrt(max(P90−elapsed,0)/P90) + randf()，再按 0.1% 格式化。用时越短，结果越高；随机扰动会造成小幅波动。

文案由 AB 分支决定

pass_text=0 默认不显示；V1 直接显示打败百分比；V2 还会按 Hard、零失误、重开/复活、上次成绩和本次百分比分层，部分分支只显示鼓励语。完成后有效的 shown_percent 会保存，用于比较下一次普通关表现。

⌁
当前必须验证普通关完成链路没有明确写入 level_config.beat_percent，所以页面底部独立的 “Top = 100 − beat” 行不能视为必现；实际展示取决于 pass_text 远程 AB。当前版本也没有在线排行榜证据。

开局“xx% 玩家”提示

普通关第 11 关起才尝试展示；默认 AB 关闭。它是本地随机激励值，不是真实玩家统计：首次尝试普通关 60—80%、Hard 40—60%；继续挑战普通关 70—90%、Hard 50—70%，按 1 位小数显示并按关卡/状态缓存。若 AB 开启 IQ 变体、上一关是 clean win、当前不是 Hard 且当前为首次尝试，则改为展示 1—6 中随机缓存的 IQ 文案，不显示百分比。

03 · Return loop

外围玩法：让玩家愿意回来

外围系统不应该抢走主玩法的注意力，而应该负责第一次理解、每天回来、完成后获得反馈和遇到问题时有出口。

3.2 Home 首页

首页必须展示

  • 当前主线关卡卡片、关卡编号、难度状态和 Play 按钮。
  • Daily 的锁定、可挑战、已完成状态和日期。
  • 连胜入口、当前连胜和可领取奖励提示。
  • 设置按钮，以及待补发奖励、AB 切换等必要弹窗。

3.4 连胜、设置和反馈

7 日连胜

当天完成符合条件的普通关或 Daily 后签到；漏签超过一天清除当前连胜。

第 7 日奖励

Hint ×2 + Locate ×2，支持直接领取或广告翻倍。

设置与反馈

Music、Sound、Vibration、Language、Coords、规则栏、How To Play、Feedback、CMP。

3.5 音效与动效

玩法反馈

进入棋盘、放置猫、错误、道具、连击、通关、失败分别使用明确音效和轻量动画。

视觉原则

猫咪和按钮负责亲和力，棋盘负责信息清晰；庆祝动效要有反馈但不能影响下一步点击。

多语言、平板、调试与性能

这些不是“最后再补”的壳层能力：它们决定产品能否被不同地区、不同设备和测试团队稳定使用。

4.4 性能与加载

当前策略

  • 优先加载 UI，避免长时间黑屏。
  • 题库按尺寸延迟加载并缓存。
  • 进入下一关前预热棋盘节点。
  • 客户端帧率上限为 60 FPS。
  • 低内存设备按配置降低广告和资源压力。

验收目标

  • 放置、撤销、提示、标记和拖动无明显卡顿。
  • 切后台、重进和关闭重开不丢状态。
  • 弹窗、广告返回和胜利动画不造成输入错位。
  • 当前版本不预加载后续五关图片/视频。
05 · Monetization

商业化：广告服务于体验

广告的目标是给玩家一个“继续、获得、复活”的选择，而不是把主玩法切碎。所有广告失败都必须有可回退路径。
P1 · 收益

5.1 广告位与触发场景

类型广告位场景用户获得
Bannerbanner玩法页展示，受门槛和设备条件控制无直接奖励
插屏start / restart / continue / sleep进入、重开、继续或回前台流程曝光
激励props_* / autox_reward道具不足或 Daily 自动标记Hint / Locate / Undo / Auto-Mark

5.2 广告策略

保护

新用户、低关卡、低内存设备和高频场景可关闭或延迟广告。

频控

插屏有展示概率、会话门槛和全局冷却，避免连续打断。

兜底

广告未 ready、初始化失败或回调失败时不阻断游戏。

5.3 奖励到账

创建奖励→广告展示→有效回调→动画中转→正式入账
↻
核心原则：不丢、不重奖励使用展示 ID 关联广告展示和回调。异常退出进入中转队列；展示后超时未回调时，按配置进入待补发奖励页，并受每日恢复次数限制。

5.4 内购范围

当前版本不包含

当前版本不提供商品列表、购买页、购买按钮、恢复购买或订阅权益。未来接入需要单独补充商品 ID、地区价格、服务端校验、权益持久化、退款和恢复购买方案。
